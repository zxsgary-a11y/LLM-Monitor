# LLM Monitor 使用说明

## 简介

LLM Monitor 是一款 Windows 桌面应用，用于实时监控 DeepSeek 等大语言模型平台的 API 用量、消费金额与账户余额。数据存储在本地 SQLite 数据库，支持自动定时同步与可视化分析。

**当前版本**: 1.0.0

## 系统要求

- Windows 10 及以上（64 位）
- 无需额外安装运行环境，绿色便携

## 安装与启动

1. 下载 llm-monitor.exe及数据库文件
2. 双击运行即可，无需安装

> 程序窗口为无边框设计（深色主题，1200×800），可通过右上角按钮最小化、最大化、关闭。

## 功能介绍

### 仪表盘

应用启动后默认进入仪表盘页面，包含以下模块：

#### 顶部状态栏
- **DeepSeek 服务状态**: 显示平台运行状态（运行正常 / 轻微异常 / 较大异常 / 严重故障 / 维护中 / 性能下降）
- **同步状态**: 显示最近一次数据同步的时间和结果
- **刷新按钮**: 手动触发 DeepSeek 服务状态与用量数据刷新

#### 统计卡片
| 卡片 | 说明 |
|------|------|
| 当前余额 | 账户剩余余额（CNY），点击可强制同步最新数据 |
| 今日费用 | 当天 API 调用累计消费 |
| 累计费用 | 所选时间范围内的总消费金额 |
| 累计 Tokens | 所选时间范围内的总 Token 用量 |
| 累计请求 | 所选时间范围内的总 API 请求次数 |

#### 年度热力图
展示全年每日 API 用量热度分布。点击某个日期格子可切换到该日的详细数据视图。

#### 图表区域（四个图表）

1. **Token 分布（饼图）**: 各模型的 Token 用量占比
2. **缓存命中 / 未命中（条形图）**: 各模型的缓存命中与未命中 Token 量，附带缓存命中率百分比
3. **模型费用（条形图）**: 各模型的费用分布，附每百万 Token 单价
4. **请求数 / 请求趋势**: 单日模式下显示各模型请求数条形图；多日模式下显示 Flash、Pro 及总请求的趋势折线图

所有图表共用时间范围选择器（今天 / 7天 / 14天 / 30天 / ALL）和日期选择器。

### 平台配置

点击左侧导航栏「平台配置」进入设置页面。

#### DeepSeek 开放平台

- **API Key**: 填入 DeepSeek API Key
- **Authorization (Bearer Token)**: 填入从 DeepSeek 平台获取的 Bearer Token
- **一键获取 Token**: 
  1. 点击「登录获取 Token」→ 在弹出的浏览器窗口中登录 DeepSeek
  2. 登录后按 F5 刷新页面
  3. 点击「提取 Token」自动填入
  4. 点击「保存配置」完成

保存配置后系统会立即执行一次数据同步，之后按设定间隔自动同步。

#### 软件设置

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 外观模式 | 深色 / 浅色主题切换 | 深色 |
| 刷新间隔 | 自动拉取数据的时间间隔 | 20 秒 |
| 首次同步范围 | 初次同步拉取多久的历史数据（修改后立即重新同步） | 1 年 |
| 数据统计范围 | 仪表盘累计卡片的统计时间范围 | — |
| 数据显示格式 | 大写（1万2千3百）/ 小写（12,300） | 大写 |
| 关闭窗口行为 | 关闭程序 / 最小化到系统托盘 | 关闭程序 |
| 悬浮窗 | 最小化到托盘时在桌面显示今日费用与 Token 用量小卡片 | 关闭 |

## 悬浮窗

启用后，当窗口最小化到托盘时，桌面会显示一个半透明悬浮窗（280×108），展示今日费用和 Token 用量概览。

悬浮窗始终置顶，不显示在任务栏中。

## 系统托盘

程序运行时会在系统托盘显示图标。右键图标可：
- 显示/隐藏主窗口
- 退出程序

## 数据存储

所有数据存储在本地 SQLite 数据库中，不上传至任何服务器。如需清除所有数据，可在「平台配置」→「关于」→「清除所有数据」中操作（不可恢复，请谨慎操作）。

## 常见问题

**Q: 配置 API Key 后没有数据？**
A: 保存配置后会自动触发首次同步（同步进度条可见）。如果长时间无数据，请检查 API Key 是否有效，或尝试点击「手动同步」。

**Q: Token 提取失败？**
A: 确保先通过「登录获取 Token」打开浏览器完成登录，按 F5 刷新后在同一个浏览器窗口下点击「提取 Token」。

**Q: 关闭窗口后程序仍在运行？**
A: 这是因为关闭行为设置为「最小化到托盘」。可在软件设置中改为「关闭程序」，或右键托盘图标退出。
